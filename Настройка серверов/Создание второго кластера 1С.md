Запуск второго кластера на портах **1640, 1641, 1660-1691**.

---

### Шаг 1. Подготовка папок для Второго кластера
Кластеры **не могут** использовать одну и ту же папку данных и временных файлов.

```bash
# 1. Создаем папку данных и временную папку для второго кластера
sudo mkdir -p /var/1c_cluster2
sudo mkdir -p /var/1c_tmp2

# 2. Выдаем права (владелец — менеджер и воркер, как в первом случае)
sudo chown -R 1c_manager:grp1cv8 /var/1c_cluster2
sudo chown -R 1c_worker:grp1cv8 /var/1c_tmp2

# 3. Вешаем магический SGID бит и права 770
sudo chmod -R 2750 /var/1c_cluster2 /var/1c_tmp2
```

### Шаг 2. Копируем настройки безопасности (`swpuser.ini`)
Чтобы второй кластер тоже умел разделять процессы `rmngr` и `rphost`, ему нужен свой файл настроек в его папке данных.

```bash
# Копируем файл из первого кластера во второй
sudo cp /var/1c_cluster/swpuser.ini /var/1c_cluster2/swpuser.ini

# Проверяем права (владельцем должен быть менеджер)
sudo chown 1c_manager:grp1cv8 /var/1c_cluster2/swpuser.ini
sudo chmod 600 /var/1c_cluster2/swpuser.ini
```

### Шаг 3. Создаем конфигурацию нового инстанса в systemd
Первый кластер у нас работает как `@default`. Второй назовем `@test`.

1.  **Создаем файл переопределения для инстанса `test`:**
    ```bash
    sudo systemctl edit srv1cv8-версия@test.service
    ```

2.  **Вставляем полный блок настроек (важно повторить Capabilities!):**
    ```ini
    [Service]
    # Работаем под тем же агентом
    User=1c_agent
    Group=grp1cv8

    # СУПЕРСИЛЫ: чтобы работал swpuser.ini
    AmbientCapabilities=CAP_SETUID CAP_SETGID
    CapabilityBoundingSet=CAP_SETUID CAP_SETGID

    # НОВЫЕ ПОРТЫ
    Environment=SRV1CV8_PORT=1640
    Environment=SRV1CV8_REGPORT=1641
    Environment=SRV1CV8_RANGE=1660:1691

    # НОВЫЕ ПУТИ
    Environment=SRV1CV8_DATA=/var/1c_cluster2
    Environment=TMPDIR=/var/1c_tmp2
    ```

### Шаг 4. Запуск и проверка

```bash
# Обновляем систему
sudo systemctl daemon-reload

# Включаем автозапуск и стартуем
sudo systemctl enable srv1cv8-версия@test.service
sudo systemctl start srv1cv8-версия@test.service
```

Проверяем, как распределились пользователи в системе:
```bash
ps aux | grep 1cv8
```

**Ты увидишь удвоенную "армию" процессов:**
*   Два `ragent` от имени `1c_agent` (на портах 1540 и 1640).
*   Два `rmngr` от имени `1c_manager` (смотрят в разные папки реестра).
*   Пачку `rphost` от имени `1c_worker`.
