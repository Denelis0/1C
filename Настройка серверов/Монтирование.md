Использование **systemd-юнитов** для монтирования — это самый современный и «чистый» способ в Linux (включая Astra Linux). Это превращает монтирование в полноценную службу, которой можно управлять через `systemctl`: запускать, останавливать и видеть статус.

Вот как это настроить по шагам.

### Важное правило именования
В systemd файл юнита должен называться **точно так же, как путь к папке**, но вместо слешей — дефисы.
*   Если папка `/mnt/1c_share`, то файл должен называться `mnt-1c_share.mount`.

---

### Шаг 1. Подготовка (папка и пароли)
Создаем файл, где будут логин и пароль, чтобы можно быть подключится к папке (обязательно права root у файла):
1. `sudo nano /root/.smbcredentials`
2. Впишите:
   ```text
   username=ЛОГИН
   password=ПАРОЛЬ
   domain=ДОМЕН
   ```
3. `sudo chmod 600 /etc/samba/имя_файла_с_данными`
4. Создайте саму папку: `sudo mkdir -p /mnt/1c_share`

---

### Шаг 2. Создание .mount юнита
Создайте файл описания службы монтирования:
```bash
sudo nano /etc/systemd/system/mnt-1c_share.mount
```

Вставьте туда следующий текст:
```ini
[Unit]
Description=Сетевая папка для 1С (Samba)
After=network-online.target
Wants=network-online.target

[Mount]
# Откуда берем (сетевой путь)
What=//IP_СЕРВЕРА/ИМЯ_ПАПКИ
# Куда монтируем (должно совпадать с именем файла!)
Where=/mnt/1c_share
Type=cifs
# Параметры (обязательно uid для прав 1С)
Options=credentials=/etc/samba/имя_файла_с_данными,iocharset=utf8,vers=3.0,uid=997,gid=1003,file_mode=0660,dir_mode=0770,nounix,noserverino,defaults,acl,user_xattr
TimeoutSec=30

[Install]
WantedBy=multi-user.target
```

---

### Шаг 3. Создание .automount юнита (рекомендуется)
Чтобы папка не «вешала» сервер при загрузке, если сеть еще не поднялась, создают второй файл — **автомонтер**. Он будет «будить» основную службу монтирования только тогда, когда кто-то реально обратится к папке.

```bash
sudo nano /etc/systemd/system/mnt-1c_share.automount
```

Вставьте туда:
```ini
[Unit]
Description=Автоматическое монтирование папки 1С

[Automount]
Where=/mnt/1c_share
# Тайм-аут простоя: размонтировать, если папкой не пользуются 5 минут (опционально)
IdleTimeoutSec=300

[Install]
WantedBy=multi-user.target
```

---

### Шаг 4. Запуск «сервиса» монтирования
Теперь активируйте ваши новые службы:

1. Перечитайте конфигурацию systemd:
   ```bash
   sudo systemctl daemon-reload
   ```

2. Включите **автомонтер** (именно его нужно ставить в автозагрузку):
   ```bash
   sudo systemctl enable mnt-1c_share.automount
   sudo systemctl start mnt-1c_share.automount
   ```

---

### Как теперь этим управлять?

Теперь вы можете работать с сетевой папкой как с обычной службой 1С:

*   **Проверить статус:**
    `systemctl status mnt-1c_share.mount`
    (Вы увидите, когда она была подключена и нет ли ошибок).
*   **Принудительно отключить:**
    `sudo systemctl stop mnt-1c_share.mount`
*   **Принудительно подключить:**
    `sudo systemctl start mnt-1c_share.mount`

### Почему это лучше, чем `/etc/fstab`?
1. **Зависимости:** Можно указать, что папка должна монтироваться **строго после** того, как поднимется сеть.
2. **Логирование:** Если папка отвалится, вы увидите причину в `journalctl -u mnt-1c_share.mount`.
3. **Отказоустойчивость:** Если сетевой сервер перезагрузится, `automount` сам переподключит папку при следующем обращении 1С к ней.

**Для вашей ERP на Linux это самый надежный «промышленный» вариант.** Теперь 1С сможет стабильно писать свои 80 ГБ логов или бэкапов в эту «службу».
